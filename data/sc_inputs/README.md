# Synthetic-control input panels

Generated by `australia/scripts/synthetic_control_input/build_sc_inputs.py`.

These four CSVs are the complete inputs needed to run a synthetic-control
analysis of Costco gas station entry on Australian retail fuel prices.

**4 treated Costco markets**: Coomera (QLD), Casuarina (WA), Perth Airport (WA),
Lake Macquarie (NSW).

## Files

### `treated_units.csv`: the 4 treated Costco markets
One row per (Costco × month). For each Costco, the treated outcome is the
**mean monthly unleaded price across all non-Costco stations within 5 km**
of that Costco's coordinates.

| Column | Type | Description |
|---|---|---|
| `costco_key` | string | Coomera / Casuarina / Perth Airport / Lake Macquarie |
| `state` | string | QLD / WA / NSW |
| `year` | int | 2018–2026 |
| `month` | int | 1–12 |
| `date` | ISO date | mid-month placeholder (15th) for plotting |
| `mean_price_cents` | float | mean unleaded price (¢/L) at non-Costco stations within 5 km |
| `n_observations` | int | number of price quotes that month |
| `n_unique_stations` | int | number of distinct stations contributing that month |

Total rows: **375** (4 Costcos × ~80–100 months each).

### `donor_pool.csv`: the candidate donor postcodes
One row per (postcode × month). Each donor unit is a postcode's mean unleaded
price across all non-Costco stations in that postcode.

| Column | Type | Description |
|---|---|---|
| `postcode` | string | 4-digit Australian postcode |
| `state` | string | NSW / QLD / WA |
| `year` | int | |
| `month` | int | |
| `date` | ISO date | mid-month |
| `mean_price_cents` | float | mean unleaded price (¢/L) at non-Costco stations in this postcode |
| `n_observations` | int | price quotes that month |
| `n_unique_stations` | int | distinct stations |

Total rows: **17,578** spanning **196 donor postcodes**.

### `treated_metadata.csv`: one row per treated Costco
| Column | Description |
|---|---|
| `costco_key` | name |

| `lat`, `lng` | coordinates |
| `treatment_date` | first appearance in state registry (proxy for opening) |
| `n_pre_months` | months of pre-treatment data |
| `n_post_months` | months of post-treatment data |
| `avg_n_unique_stations` | average number of distinct stations contributing to the treated mean per month |

### `donor_metadata.csv`: one row per donor postcode
| Column | Description |
|---|---|
| `postcode` | 4-digit code |
| `state` | NSW / QLD / WA |
| `avg_unique_stations` | average distinct stations per month |
| `n_months_observed` | months with any observations |
| `min_dist_any_costco_km` | average across stations of min distance to any Costco |

## Filtering applied to donor pool

A postcode was **excluded** if any of:
- It lies within 20 km of any Costco (362 postcodes excluded; could be picking up Costco effects)
- Fewer than 3 unique stations average per month (446 excluded; monthly means too noisy)
- Fewer than 24 months of observations (0 excluded; coverage was good)

196 donor postcodes survived all filters: NSW = 104, QLD = 56, WA = 36.

## How to use these in synthetic control

Standard workflow with `pysyncon` or `SyntheticControlMethods` (Python):

```python
import pandas as pd

treated = pd.read_csv("treated_units.csv")
donors  = pd.read_csv("donor_pool.csv")
treated_meta = pd.read_csv("treated_metadata.csv")

# Pivot to wide format that SC libraries expect (rows=time, cols=units)
treated_wide = treated.pivot(index="date", columns="costco_key",
                              values="mean_price_cents")
donor_wide   = donors.pivot(index="date", columns="postcode",
                              values="mean_price_cents")

# Run synthetic control once per Costco, restricting donors to same state:
for ckey, meta in treated_meta.set_index("costco_key").iterrows():
    state = meta["state"]
    treatment_date = meta["treatment_date"]
    state_donors = donors[donors["state"] == state]["postcode"].unique()
    # ... fit synthetic weights on pre-period, project counterfactual, etc.
```

## Frequency of fit

We recommend running synthetic control **separately for each of the 4 Costcos**
(rather than pooling them), then aggregating effects. Reasons:

1. Each Costco's treated trajectory is distinct (different pre-trends, different
   metro), so a single pooled fit would average over real heterogeneity.
2. Per-Costco fits enable per-Costco placebo tests, which is the standard way
   to assess statistical significance with synthetic control.
3. The spec's example (Anglou, Sanders & Stamatopoulos 2025) does exactly this:
   one synthetic-control fit per treated state, then aggregates.
